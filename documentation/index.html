# Introduction

Forme is an MVC application framework built on top of WordPress. You can use it to craft themes and plugins with a modern expressive php syntax.

# Why Forme?

We've got a lot of love for WordPress - it's got comprehensive docs, it powers half the internet, and many web developers earn their php chops working with it. Clients love it - its ubiquity and familiarity makes it the defacto CMS of choice for a vast amount of projects, from mom and pop outfits right up to enterprise level.

However, coding for it can feel like being stuck in a bygone era of clunky workflows, bad procedural patterns, weird antiquated syntax, and a tendency towards unmaintainable spaghetti awfulness. And don't even get us started on the loop, which is just... weird. 🤢

It can feel like taking a big step backwards, especially if you've done any amount of work in modern PHP frameworks such as Laravel or Symfony.

Forme aims to solve these pain points.

There are a bunch of really cool Open Source projects already operating in this space, including LumberJack (opens new window), Timber (opens new window), Themosis (opens new window) and the Roots (opens new window) ecosystem. We've taken inspiration and ideas from all of these, but built Forme to follow our own specific and opinionated workflow.

# Features

  • Keep things DRY with a typical MVC pattern
  • Eloquent ORM and Phinx DB Migrations
  • League Plates 4 views (but Plates 3, Blade or Twig also supported)
  • WordPress templates become controllers.
  • DI Container
  • Flexible routing system
  • YAML config for WordPress hooks
  • Registry pattern
  • Bootstrap 5
  • PSR-4 and Composer
  • Dotenv
  • PSR-7 Requests and Responses
  • PSR-15 Middleware
  • Debugging Tools
  • A full featured code generation cli for easy scaffolding

# What Forme is not a good fit for

Since Forme relies on Composer/PSR-4 and a custom server setup, publishing plugins or themes to the WordPress store is not currently practical. Better to stick with distributing them via packagist for the time being. We are working on a solution though, so watch this space!
