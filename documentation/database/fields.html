# Fields

Forme is tightly integrated with Advanced Custom Fields (opens new window), and includes simple conventions and some useful magic for handling fields.

# Creating a field group

Field groups live in app/Fields and should extend Forme/Fields/FieldGroupInterface. They have a single method, add, into which you can copy the php from the export feature in ACF.

// app/Fields/FooBarFielGroup.php
declare(strict_types=1);

namespace YourNameSpace\YourApp\Fields;

use Forme\Framework\Core\FieldGroupInterface;

final class FooBarFieldGroup extends FieldGroupInterface
{
    public function add(): void
    {
        acf_add_local_field_group([
            'key'                   => 'group_5fc6297c4b8cb',
            'title'                 => 'Foo',
            'fields'                => ['etc'],
        ]);
    }
}

TIP

Don't put more than one field group per class. Also try to keep field groups as small as possible when you're configuring your forms in acf itself. You want to keep things composable and maintainable.

# Field Registry

The field registry class takes care of adding all your field groups, it simply globs the classes in app\Fields on the acf/init hook, and cycles through each one, calling add.

# Automatic injection into Template Controllers

ACF Fields for the queried object are automatically injected into Template Controllers - take a look at $args['fields'].

Otherwise just call get_field or get_fields as you would do normally.

TIP

You probably won't need to use the oddly redundant get_sub_field function, especially since you should not be using the have_rows antipattern in the first place. Try to refactor if you start reaching for those.

# Code Generation

You can easily create a new field group via the cli:

forme make field FooBar

This will make a new field group class YourNameSpace\YourApp\Fields\FooBarFieldGroup ready to populate.
