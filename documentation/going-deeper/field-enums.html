# Field Enums

Advanced Custom Fields (opens new window) is an awesome plugin which enables you to easily add custom fields to any type of WordPress post. WordPress developers involved in custom built plugins or themes will most likely have come across it, and Forme itself is closely integrated with it.

You get an easy to use UI to configure your fields. This is great for scaffolding stuff out, but our approach is to then always convert this field group set up to php code by enclosing it one of our FieldGroup classes. This means the configuaration is fixed in version control rather than being part of database content, which is inherently brittle.

One limitation we have found though is that if you then need to reference your fields elsewhere in the codebase, this then encourages the use of magic values (opens new window), which quickly becomes unmaintainable. You also miss out on having your editor being able to autocomplete your field names, keys and labels.

We've come up with a Field Enum pattern as a way to solve this problem. In essence, this simply replaces the magic strings with typed enum properties which we are available at compile time. Apart from reducing unstable copy pasta hell, it also means that your editor can now autocomplete for you, and you get type safety for free.

We've got an abstract AcfEnum class which you can extend for this purpose.

<?php
declare(strict_types=1);

namespace Foobar\ProjectNameSpace\Enums;

use Forme\Framework\Enums\AcfEnum;

/**
 * @method static self FOO()
 */
final class FooBarField extends AcfEnum
{
    protected $fieldNames = ['field_EJF95fA9n' => 'foo'];

    protected static function values(): array
    {
        return ['FOO' => 'field_EJF95fA9n'];
    }

    protected static function labels(): array
    {
        return ['FOO' => 'Foo Field'];
    }
}

Then in your ACF field group definition, you can now replace the magic strings with the enum:

<?php
...
    'fields' => [
        [
            'key' => FooBarField::FOO()->key,
            'label' => FooBarField::FOO()->label,
            'name' => FooBarField::FOO()->name,
            'type' => 'text',
            'instructions' => '',
            'required' => 0,
            'conditional_logic' => 0,
            'wrapper' => [
                'width' => '',
                'class' => '',
                'id' => '',
            ],
            'default_value' => '',
            'placeholder' => '',
            'prepend' => '',
            'append' => '',
            'maxlength' => '',
        ],
    ],
...

Our AcfEnum class uses Spatie's Enum under the hood, but we've added some custom properties for the key, label and name of the field. You can use this same syntax throughout your codebase.

# Code Generation

The above manual process can quickly become very laborious and error-prone for larger field groups, so we've added a cli command to automate this.

forme make field-enum FooBar

Just follow the instructions to select the Field Group you want to generate an enum for, and choose the enum property names. The cli will generate a new enum class for you, as well as find and replace all the magic strings in the field group definition.

WARNING

This command works fine for fairly straight forward use cases, but might struggle with nested field groups and repeaters. YMMV.
